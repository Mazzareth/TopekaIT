Topeka IT Portal Project State

The Topeka IT Portal is a role-based operations application for division workers, supervisors, IT staff, and central administrators. It brings requests, printer health, device assignments, lockers, loaners, user access, and selected serial-gateway readings into one controlled portal.

4 workspaces Worker, Supervisor, IT/Admin, and Super Admin views.
Live ops focus Queues, health, assignments, and exceptions are front and center.
Division-ready Each division has its own operating data and settings.
Review ready Best next step is a guided walkthrough with sample real data.
Current state: the application has implemented user-facing workflows for request tracking, printer monitoring, asset management, locker and loaner handling, admin reporting, and Lantronix device polling. This handout describes the application as of May 8, 2026 from the current project files.

Access Model In place

Permissions are grouped by work area: workspaces, tickets, printers, assets, lockers, users, and admin/system tools.

  • Users inherit defaults from their role level.
  • Specific permissions can be allowed or denied per user.
  • Navigation only shows tools a user can access.

Division Model In place

The app separates central administration from division work so each division can keep its own users, assets, printers, tickets, and settings.

  • Super Admins can enter a division from the Admin area.
  • Division settings include printer password code and ZIP fields.
  • Cross-division reporting stays in the Admin workspace.

Core Application Features

Current user-facing capability

Ticket and Request Management Active

Workers and managers can submit requests. IT gets a queue designed for daily work, not just viewing records.

  • Ticket filters for Open, In progress, On hold, Resolved, and All.
  • Priority is visible in the list and urgent/high items are emphasized.
  • Tickets can be assigned, moved through statuses, and closed with resolution notes.
  • Tickets can be linked to a printer or asset.
  • Repair and RMA asset statuses can automatically create IT tickets.
Main views: IT Ticket Queue, Worker My Requests, Supervisor Requests

Asset Management Active

The asset console tracks devices by category, assignment, status, and health.

  • Categories include SAE devices, TC77 PODs, scanners, and batteries.
  • Batteries are quantity-aware instead of serial-only.
  • Assets can be searched by tag, serial number, IMEI, model, or holder.
  • Scanning support can match tags, serials, IMEIs, and IDs.
  • Status and issue flags support attention views for repair, RMA, missing, hold, and investigation states.
Main views: Asset Console, Asset Detail, Add Asset

Printer Operations Active

Printer pages are health-first, with attention on down printers, warnings, latency, and current alerts.

  • Workers and managers can view printer status and report a printer issue.
  • IT can add, edit, and remove printers when permitted.
  • Printer models are catalog-backed, with T8000 as the default model.
  • Auto setup supports test-first bulk discovery and run-all configuration.
  • Printer error logs keep friendly messages plus raw detail when needed.
Main views: Printer Status, Printer Admin, Printer Detail

User and Access Management Active

User management covers account basics plus operational fields needed by supervisors and IT.

  • Accounts track name, username, position, division, locker number, audit status, and last active time.
  • Admins can reset passwords when allowed.
  • Role tiers control default access.
  • The access catalog allows per-user permission overrides.
Main views: Users, Workers, Access Catalog
Topeka IT Portal Core Features

Supervisor and IT Workspaces

Daily operating screens

Supervisor Home

A shift workspace for employee lookup, gear handoff, printer issues, and crew requests.

  • Employee search by name.
  • Locker number, combo reveal, assigned device, and device status.
  • Action buttons for lockers, loaners, assets, printers, requests, and new request creation.

Locker Console

A visual locker board for floor lockers and control center lockers.

  • Floor lockers 1-90 and control center lockers 110-115.
  • Open, assigned, and attention states at a glance.
  • Locker detail shows employee, combo access, lock serial, assigned device, notes, and audit checkbox.

Loaner Console

Tracks the spare pool and active sign-outs.

  • Active loans show borrower, device, reason, duration, date loaned, and overdue status.
  • Spare pool shows available and loaned units.
  • Managers can issue a spare or mark one returned when permitted.

Devices Needing Attention

Supervisor home highlights devices in RMA, missing, repair, investigation, or holding states.

  • Shows the device, model, holder, status, and how long the device has been in that state.
  • Provides a quick path to issue a spare when the user has permission.
  • Links back to the full asset console for deeper review.

IT Control Room

IT gets a high-signal operations dashboard across tickets, printers, assets, and recent activity.

  • Scope controls for 1 hour, 24 hours, 7 days, and 30 days.
  • Open ticket count, urgent/high priority counts, printer attention, assets out, and average close time.
  • Printer fleet heatmap, top-of-queue list, and recent activity feed.

Admin, Reporting, and Device Monitoring

Central oversight

Admin Reports

The Admin area includes a Reports tab that currently focuses on printer errors across divisions.

  • Report selector with division filter.
  • All Printer Errors report groups events by alert type.
  • Expanded groups show recent occurrences, division, department, raw message, and a View action.
  • View action can enter the correct division and open the printer detail page.

Division Administration

Central admins can create and manage division records used by the portal.

  • Division list with code and ZIP fields used by printer setup.
  • Save action for division settings when permitted.
  • Enter action for switching into a division workspace.

Lantronix Device Monitoring

The portal can show configured Lantronix serial gateways and poll fuel inventory readings.

  • Device list with endpoint, division, and reporting status.
  • Current metrics for volume, height, water, and temperature.
  • Tank volume chart for the last 30 days when enough samples exist.
  • Recent poll history with success/failure status and raw response when available.

Background Monitoring

Background services support the operational views.

  • Printer health is checked on a recurring cadence.
  • Ping history powers printer latency and trend displays.
  • SNMP trap and printer log ingestion support alert and error views.
  • Lantronix polling stores samples for charts and recent readings.
